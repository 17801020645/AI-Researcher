--------------------------------------------------------------------------------

✨ AI-Researcher 系统支持两个不同层级的用户输入查询 ✨

**Level 1：详细想法描述**
<br/> 在此层级，用户提供其具体研究想法的全面描述。系统处理这些详细输入，并基于用户的明确要求制定实现策略。

**Level 2：基于参考文献的创意生成**
<br/> 这一更简单的层级允许用户在不具备具体想法的情况下提交参考论文。用户查询通常遵循如下格式："我有一些参考论文，请基于这些论文提出一个创新想法并实现它。" 系统随后分析所提供的参考文献，生成并发展新颖的研究概念。

## ⚡ 快速开始

<span id='installation'/>

### 安装

#### AI 环境安装

1. 使用 [uv](https://docs.astral.sh/uv/)

> 我们建议使用 [uv](https://docs.astral.sh/uv/) 管理项目依赖（比 conda 快得多）

```bash
# install uv
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.bashrc

# clone the project
git clone https://github.com/HKUDS/AI-Researcher.git
cd AI-Researcher

# install and activate enviroment
uv venv --python 3.11
source ./.venv/bin/activate
uv pip install -e .
playwright install
```

#### Docker 安装

为搭建智能体交互环境，我们使用 Docker 进行容器化。请确保您的系统已安装 [Docker](https://www.docker.com/) 后再继续。运行研究智能体时，我们使用 Docker 镜像 `tjbtech1/airesearcher:v1`。您可以通过以下命令拉取该镜像：

```bash
docker pull tjbtech1/airesearcher:v1
```

或者，您也可以根据我们提供的 [Dockerfile](../docker/Dockerfile) 自行构建 Docker 镜像。

```bash
cd ./docker && docker build -t tjbtech1/airesearcher:v1 .
```

<span id='api-keys-setup'/>

### API 密钥配置

基于提供的 `.env.template` 文件创建环境变量配置文件。在该文件中，您需要设置包括 API 密钥、测试用例 instance id 等在内的配置。

```bash

# ================ container configuration ================
# workplace of the research agent
DOCKER_WORKPLACE_NAME=workplace_paper
# base image of the research agent
BASE_IMAGES=tjbtech1/airesearcher:v1
# completion model name, configuration details see: https://docs.litellm.ai/docs/
COMPLETION_MODEL=openrouter/google/gemini-2.5-pro-preview-05-20
# cheep model name, configuration details see: https://docs.litellm.ai/docs/
CHEEP_MODEL=openrouter/google/gemini-2.5-pro-preview-05-20
# specific gpu of the research agent, can be: 
# '"device=0"' using the first gpu
# '"device=0,1"' using the first and second gpu
# '"all"' using all gpus
# None for no gpu
GPUS='"device=0"'
# name of the container
CONTAINER_NAME=paper_eval
# name of the workplace
WORKPLACE_NAME=workplace
# path of the cache
CACHE_PATH=cache
# port of the research agent
PORT=7020
# platform of the research agent
PLATFORM=linux/amd64

# ================ llm configuration ================
# github ai token of the research agent
GITHUB_AI_TOKEN=your_github_ai_token
# openrouter api key of the research agent
OPENROUTER_API_KEY=your_openrouter_api_key
# openrouter api base url of the research agent
OPENROUTER_API_BASE=https://openrouter.ai/api/v1

# ================ task configuration ================
# category of the research agent, based on: ./benchmark/final. Can be: 
# diffu_flow
# gnn
# reasoning
# recommendation
# vq
# example: ./benchmark/final/vq
CATEGORY=vq
# instance id of the research agent, example: ./benchmark/final/vq/one_layer_vq.json
INSTANCE_ID=one_layer_vq
# task level of the research agent, can be: 
# task1
# task2
TASK_LEVEL=task1
# maximum iteration times of the research agent
MAX_ITER_TIMES=0
```

### 示例 2（类别：向量量化）

<details>
  <summary><b>输入：Prompt</b><br><p>我有一些参考论文，请基于这些论文实现以下想法：</p><ol>
    <li>所提模型聚焦于离散表示学习，用于图像生成、深度估计、上色和分割等任务，并将所提方法集成到自回归 Transformer 等架构中。</li>...</ol></summary>
<div>
  <!-- <p>I have some reference papers, please implement the following idea with these papers:</p> -->
  <ol start="2">
    <li>核心技术：
      <ul>
        <li><strong>简化量化</strong>：使用标量量化替代 VQ 的简化量化方法。</li>
        <li><strong>维度投影</strong>：定义函数将编码器输出投影到可管理的维度（通常为 3 到 10）。</li>
        <li><strong>梯度传播</strong>：实现 Straight-Through Estimator（STE）以支持梯度通过量化操作传播。</li>
      </ul>
    </li>
    <li>技术组件：
      <ul>
        <li><strong>边界函数</strong>：压缩数据维度并将值限制在期望范围内。使用类似 \(f(z) = \left\lfloor \frac{L}{2} \right\rfloor \tanh(z)\) 的函数投影数据，其中 \(L\) 为量化级数。</li>
        <li><strong>量化过程</strong>：将每个有界维度四舍五入到最近整数，得到量化输出。</li>
        <li><strong>损失函数</strong>：在 VAE 典型的重建损失范式下运行，以优化所提模型参数。</li>
      </ul>
    </li>
    <li>实现细节：
      <ul>
        <li><strong>关键参数</strong>： 
          <ul>
            <li>维度数 \(d\) 和每维级数 \(L\) 应根据目标复现的码本大小定义（例如，对所有 \(i\) 设置 \(L_i \geq 5\)）。</li>
          </ul>
        </li>
        <li><strong>输入/输出规格</strong>： 
          <ul>
            <li>边界函数的输入为最终编码器层的输出；量化后输出格式为 \(\hat{z}\)，形状与原始 \(z\) 一致。</li>
          </ul>
        </li>
        <li><strong>约束</strong>： 
          <ul>
            <li>确保所有输入经过充分预处理，处于边界函数的有效范围内。</li>
          </ul>
        </li>
      </ul>
    </li>
    <li>逐步集成：
      <ul>
        <li><strong>步骤 1</strong>：训练标准 VAE 模型并获得编码器输出 \(z\)。</li>
        <li><strong>步骤 2</strong>：对 \(z\) 应用边界函数 \(f\)，将输出维度限制为可用值。</li>
        <li><strong>步骤 3</strong>：使用四舍五入过程对边界化后的 \(z\) 进行量化，生成 \( \hat{z} \)。</li>
        <li><strong>步骤 4</strong>：将原始 \(z\) 与 \(\hat{z}\) 结合重建损失，通过 STE 进行梯度计算并反向传播。</li>
      </ul>
    </li>
    <li>关键实现细节：
      <ul>
        <li>确保四舍五入过程可正确求导；使用 STE 在反向传播中维持梯度流。</li>
        <li>通过经验试验选择最优维度与级数以维持高码本利用率，并监控性能以在需要时调整参数。</li>
        <li>根据本文给出的结构配置模型（epoch 数、batch size 等），确保超参数与所提方法集成建议一致。</li>
      </ul>
    </li>
  </ol>

## ✨ AI-Researcher 工作原理

* 🔄 **端到端科学研究自动化系统**
  <br>我们的 **AI-Researcher** 通过集成流水线为完整科学研究生命周期提供全面自动化。系统在三个战略阶段编排研究活动：
  1. **文献综述与创意生成** 📚💡
     * 🔍 **资源收集器**：系统性地从主要学术数据库（如 arXiv、IEEE Xplore、ACM Digital Library 和 Google Scholar）、代码平台（如 GitHub、Hugging Face）以及各科学领域的开放数据集中自动收集全面的研究材料。
       
     * 🧠 **资源筛选器**：通过质量指标（如引用数、代码维护状况、数据完整性）和相关性评估，筛选高影响力论文、维护良好的代码实现和基准数据集。
     
     * 💭 **创意生成器**：基于已识别的研究资源（包括高影响力论文和代码仓库），创意生成器通过全面分析系统性地构建新颖研究方向。它自动评估当前方法局限、映射新兴技术趋势，并探索未开垦的研究领域。
  
  2. **新算法设计、实现与验证** 🧪💻
     <br>**设计 → 实现 → 验证 → 优化**
     * 📝**设计阶段**：初始阶段聚焦概念开发，在此构思新颖算法想法并建立理论基础。此阶段我们仔细规划实现策略，确保所提方案在保持实际可行性的同时超越现有方法。
     
     * ⚙️**实现阶段**：将抽象概念转化为具体代码实现。此阶段涉及开发功能模块、建立稳健的测试环境，以及创建实验验证所需的基础设施。
  
     * 🔬**验证阶段**：系统性实验构成验证过程的核心。我们执行全面测试以评估算法性能、收集指标并记录所有发现。此阶段确保严格验证实现与实际需求的一致性。
       
     * 🔧**优化阶段** 🔬：基于验证结果，进入迭代优化循环。此阶段涉及识别改进空间、优化代码效率并实施必要增强。我们仔细分析性能瓶颈，并为下一轮开发迭代规划战略改进。
  
  3. **论文撰写** ✍️📝
     * **Writer Agent** 📄：通过整合研究想法、动机、新设计的算法框架和算法验证性能，自动生成完整学术论文。采用分层写作方法，生成精确清晰的成稿。

🚀 这一完全自动化系统消除了整个研究生命周期中的人工干预需求，实现从初始概念到最终发表的无缝科学发现。🚀 它是出色的研究助手，帮助研究人员高效达成目标。

--------------------------------------------------------------------------------

* 🔬 **全面基准测试套件**
  <br>我们开发了全面且标准化的评估框架，以客观评估 AI 研究者的学术能力及其学术工作质量，并集成多项关键创新以确保深入且可靠的评估。

  1. 👨‍🔬 **专家级标准答案**：基准利用人类专家撰写的论文作为标准参考，建立高质量比较与验证基准。

  2. 🌈 **多领域覆盖**：我们的基准全面涵盖 4 大研究领域，确保广泛适用性：计算机视觉（CV）、自然语言处理（NLP）、数据挖掘（DM）和信息检索（IR）。

  3. 🌐 **完全开源的基准构建**：我们完全开源了基准构建的方法与流程，包括处理后数据集、数据收集流水线和处理代码的完整访问权限。这确保了**评估透明性**，同时赋能社区为测试 AI 研究者定制和构建特定领域的基准。
    
  4. 📊 **全面评估指标**：我们的评估框架采用分层系统化方法，任务按想法提供程度分为两个层级。借助专门的 **Evaluator Agents**，框架在多个维度进行深入评估，确保稳健且全面的评价。关键评估指标包括：1）**新颖性**：评估研究工作的创新性与独特性。2）**实验全面性**：评估实验设计、执行与严谨性。3）**理论基础**：衡量理论背景与基础的强度。4）**结果分析**：分析结果解读的深度与准确性。5）**写作质量**：审查书面报告的清晰度、连贯性与结构。

🚀 **推进研究自动化**。该基准套件为评估研究自动化能力提供客观框架，设计为持续演进，纳入新进展并扩展范围以满足研究社区不断增长的需求。

--------------------------------------------------------------------------------

* 🌟 **易于使用的 AI 研究助手**
  <br>**AI-Researcher** 为研究自动化提供真正流畅且易用的体验，使用户能够专注于创新而无需技术障碍。主要特性包括：
  
  1. 🌐 **多 LLM 提供商支持**：轻松集成 Claude、OpenAI、Deepseek 等领先语言模型提供商。研究人员可选择最适合其需求的 AI 能力。
  
  2. 📚 **轻松启动研究**：以无与伦比的便捷性开启研究之旅！只需提供相关论文列表，**AI-Researcher** 即可处理其余工作——无需上传文件、贡献初始想法或应对复杂配置。它是高效启动研究流程的得力工具。
  
  3. 🧠 **最低领域专业知识要求**：AI-Researcher 通过自主识别关键研究空白、提出创新方法并执行完整研究流水线来简化研究过程。虽然一定的领域理解可以提升效果，但该工具旨在帮助各专业水平的用户也能轻松取得有意义的研究成果。
  
  4. 📦 **开箱即用**：从一开始就体验流畅的研究自动化。AI-Researcher 仅需少量设置即可使用，即时访问高级能力。无需繁琐配置，即可轻松高效地加速研究进程。
